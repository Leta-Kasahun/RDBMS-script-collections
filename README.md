SQL (Structured Query Language) is the cornerstone of working with relational databases. While MySQL is widely adopted for web applications and open-source projects, Oracle offers robust enterprise capabilities, including PL/SQL, RAC, and Data Guard.

This repository provides a side-by-side SQL experience ‚Äî learn how MySQL and Oracle compare in syntax, features, and behavior.

## ‚úçÔ∏è Basic CRUD Operations

| Operation | MySQL Example | Oracle Example |
|----------|---------------|----------------|
| Create   | `CREATE TABLE users (id INT AUTO_INCREMENT, name VARCHAR(50));` | `CREATE TABLE users (id NUMBER GENERATED BY DEFAULT AS IDENTITY, name VARCHAR2(50));` |
| Insert   | `INSERT INTO users (name) VALUES ('Alice');` | Same syntax |
| Read     | `SELECT * FROM users;` | Same syntax |
| Update   | `UPDATE users SET name='Bob' WHERE id=1;` | Same syntax |
| Delete   | `DELETE FROM users WHERE id=1;` | Same syntax |

---

## üèóÔ∏è DDL & Constraints

- Use `CONSTRAINT` for primary/foreign keys in both.
- Oracle supports `DEFERRABLE INITIALLY DEFERRED` for constraints.
- MySQL auto-increments with `AUTO_INCREMENT`; Oracle uses sequences.

---

## üîÑ Joins & Subqueries

- Both support `INNER`, `LEFT`, `RIGHT`, and `FULL OUTER JOIN`.
- Subqueries work in both but Oracle handles inline views with more optimizer control.

---

## üóÇÔ∏è Data Types & Differences

| Data Type | MySQL | Oracle |
|-----------|-------|--------|
| Text      | `VARCHAR`, `TEXT` | `VARCHAR2`, `CLOB` |
| Number    | `INT`, `DECIMAL` | `NUMBER(p,s)` |
| DateTime  | `DATETIME`, `TIMESTAMP` | `DATE`, `TIMESTAMP` |

üìà Indexing & Optimization
